# 0155-Polymorphism Cheatsheet

- **Purpose**: allow objects of different types to be used interchangeably if they implement the same interface/methods.
- **Duck typing**: "if it quacks like a duck"; no explicit interface needed.
- **Inheritance polymorphism**: subclasses override parent methods; treat them as parent type.
- **Example style**: define `speak()` or `draw()` on multiple classes.
- **Check**: use `isinstance` only when necessary; prefer common method names.
