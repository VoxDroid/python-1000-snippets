# 0163-Factory-Pattern Cheatsheet

- **Purpose**: encapsulate object creation in a separate 'factory' class or method.
- **Examples**: conditional logic, mapping type names to classes, registration.
- **Benefits**: decouples clients from concrete classes; simplifies extension.
- **Variations**: simple factory, factory method (subclasses override creation), abstract factory.
- **Tips**: use static or class methods; avoid large `if` chains by using dict lookup.
