# 0169-Decorator-Pattern Cheatsheet

- **Purpose**: add responsibilities to objects dynamically without modifying their class.
- **Mechanism**: create decorator classes that hold a reference to the original object and implement the same interface.
- **Usage**: wrap an object with multiple decorators to compose behaviors.
- **Compare**: not to be confused with Python function decorators; this is the OOP design pattern.
- **Tips**: ensure decorators forward all needed methods; keep each decorator small and focused.
