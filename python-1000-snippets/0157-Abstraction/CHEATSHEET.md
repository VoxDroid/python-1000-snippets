# 0157-Abstraction Cheatsheet

- **Purpose**: define interfaces and abstract behavior without implementation using `abc` module.
- **Create abstract class**: subclass `ABC` and mark methods with `@abstractmethod`.
- **Cannot instantiate** abstract classes until all abstracts are implemented.
- **Use for** enforcing method signatures; helpful for large codebases or plugins.
- **Alternative**: use informal protocols or duck typing if strict enforcement unnecessary.
